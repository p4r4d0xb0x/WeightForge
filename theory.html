Ferry / 이론

외부 데이터 없이 합성 probe만으로, Teacher와 Student가 같은 답을 내도록 보장하는 것이 언제 가능한가 — 그리고 그 보장의 정확한 조건은 무엇인가.

엔지니어용 판본 · 각 단계의 수식·알고리즘 절차·실측치를 함께 제시한다.

§0–11 toy core(같은-답 보장의 조건) · §12–15 실모델 확장 (Gemma-2 → Aster: 도달 vs 정렬의 정직한 경계)

이후 모든 단계는 같은 기호를 쓴다. 먼저 기호와 "같은 답" 목표를 수식으로 못 박는다.

Ferry가 풀려는 목표는 "두 모델이 모든 입력에서 같은 답"이다. 이를 함수 동치(functional equivalence)로 적으면:

목표(이상): fS(x) = fT(x) ∀ x 목표(측정가능): 𝔼x∼P‖ fS(x) − fT(x) ‖² → 0 분류 관점: argmax fS(x) = argmax fT(x) (top-1 일치)

데이터셋 라벨이 없으므로 "정답"은 항상 teacher의 출력 fT(x)다. 그래서 외부 데이터 없이도 목표가 정의된다 — 이것이 Ferry의 data-free 설계 핵심이다. agreement가 보고하는 세 수치 mse/top1_agree/cosine가 위 세 식의 실측 추정치다.

1두 단계: 가중치 전이 → 출력 정합

Ferry는 두 단계로 같은 답을 만든다. 1단계는 가중치 공간에서 teacher 텐서를 student로 옮기고(초기화), 2단계는 합성 probe로 측정한 teacher 출력에 student의 마지막 선형층을 닫힌 형식 최소제곱으로 맞춘다.

2단계 정합이 끝나면 student 출력이 teacher 위로 포개진다 — 데모 실측(깊은 선형 teacher MLP[32,128,96,64,10]) held-out top-1 100%, MSE≈8e-15 (단, 아래 용량 조건 충족 + teacher가 선형일 때).

전체 파이프라인을 단계별로 적으면(LM이 아니면 stage 0 생략):

  1. stage 0 — 어휘 정합 (LM 한정): 두 모델의 vocabulary가 다르면 VocabMap으로 teacher logits를 student 어휘 공간으로 사상한다 (§9).
  2. stage 1 — 가중치 전이: teacher 텐서를 student로 결정적 변환해 옮긴다 → student를 teacher 근처로 초기화(warm start)한다 (§4). θS ← Transfer(θT, θSinit)
  3. stage 2 — 출력 정합: 합성 probe로 측정한 teacher 출력에 student의 마지막 선형층(필요시 은닉층)을 닫힌 형식 최소제곱으로 맞춘다 (§5·§7).
  4. stage 3 — gradient distillation: 남은 잔차를 매 step 새 합성 probe로 gradient fine-tune해 닫는다 (§10). 데이터는 끝까지 쓰지 않는다.

stage 1–2는 전부 결정적 대수(복사·SVD·lstsq)라 gradient·데이터가 없고, gradient는 stage 3에만 격리된다. stage 1이 좋은 출발점을 주므로 stage 2·3이 빠르게 수렴한다.

2왜 가능할법한가 — 저차원 구조 가설

학습된 가중치 행렬은 보통 소수의 특이방향(singular direction)에 에너지가 몰린다. 그렇다면 차원이 줄어든 student로 주요 방향만 투영해도 본질이 살아남는다. 이것이 SvdProject의 전제다.

임의의 행렬 W ∈ ℝm×n은 특이값 분해(SVD)로 쪼갤 수 있다. 특이값 σ1≥σ2≥…≥σr가 각 방향의 "에너지"다:

SVD: W = U Σ V⊤ = Σi=1..r σi ui vi⊤ top-k 근사: Wk = Σi=1..k σi ui vi⊤ (Eckart–Young 최적 rank-k 근사) 보존 에너지: E(k) = (Σi=1..k σi²) / (Σi=1..r σi²) ∈ [0, 1]

E(k)가 작은 k에서 이미 1에 가까우면(=특이값이 급감하면) 차원을 k로 줄여도 정보 손실이 작다. SvdProject는 정확히 Wk를 student 차원에 맞춰 쓴다.

대상top-8 에너지top-16 에너지top-32 에너지해석
데모 teacher net.1.weight (96×128, random) 24.3%42.6%68.7% 완만

실측: python ferry.py의 깊은 teacher MLP[32,128,96,64,10] 중 net.1.weight 특이값은 1.09→0.10으로 완만하게 감소한다. top-32 방향(128 중)을 살려도 에너지의 ~69%만 보존된다 — 랜덤 init에서는 저차원 투영의 이득이 제한적이다. 학습된 모델일수록 왼쪽 곡선에 가까워져 Ferry의 전제가 성립한다 — 이것이 "가능 조건"이다.

3치환 불변성 — 직접 복사가 위험한 이유

두 망의 같은 위치 뉴런이 같은 역할을 한다는 보장은 없다. 순서만 뒤섞여도 함수는 동일하다(permutation invariance). 그래서 이름·위치 기반 단순 복사는 어긋난 짝을 이어붙일 수 있다.

현재 Ferry는 이름 기반 매칭까지만 한다(match_tensors). permutation 정렬은 아직 없음 → 알려진 한계, 향후 Git Re-Basin 식 정렬로 보완 가능.

모든 텐서는 shape만 보고 4가지 결정적 변환 중 하나로 분기한다. 학습·데이터·최적화 없음 — 순수 대수.

kind조건연산정보 손실

각 분기의 연산을 수식으로 적으면 (teacher 텐서 WT ∈ ℝm×n, student 목표 shape p×q):

Copy: WS = WT (shape 동일, m=p · n=q) CropPad: WS[i,j] = WT[i,j] if i<min(m,p), j<min(n,q); else 0 SvdProject: WT = UΣV⊤, k = min(m,n,p,q), WS = crop/pad( U:k Σ:k V:k⊤ ) → p×q (Eckart–Young top-k) Skip: WS = WSinit (rank 불일치 → 미전이)

분기는 오직 shape로 결정되므로 결정적·재현가능하다. CropPad는 잘려나간 부분의 정보를, SvdProject는 하위 특이방향 σk+1…σr의 에너지를 잃는다(§2의 E(k)가 그 손실 척도). 전이 직후 답이 틀려도 정상 — stage 2가 마지막 층을 다시 맞춘다.

5출력 정합 — 같은 답을 만드는 닫힌 형식 해

합성 probe X를 teacher에 흘려 목표 출력 Y=teacher(X)를 얻고, student의 마지막 선형층 입력 특징 F를 잡는다. 그 다음 [F|1] · W = Y를 최소제곱(lstsq)으로 한 번에 푼다. gradient 루프 없음.

이 해는 probe뿐 아니라 임의의 입력에 대해 student를 teacher와 일치시킨다 — student 마지막 표현이 teacher 출력을 선형 재구성할 수 있다면.

5.1 절차와 수식

  1. probe 통과·특징 포착: 합성 probe X ∈ ℝn×d를 student에 흘리고, 마지막 선형층의 입력 특징 F ∈ ℝn×mS를 forward hook으로 잡는다. 동시에 목표 Y = fT(X) ∈ ℝn×c를 teacher에서 구한다 (c = 출력 차원).
  2. 증강행렬 구성: 편향을 흡수하기 위해 1열을 덧붙인다. F̃ = [ F | 𝟏 ] ∈ ℝn×(mS+1), W = [ Wᵀ ; b ] ∈ ℝ(mS+1)×c
  3. 최소제곱 해: 다음을 닫힌 형식으로 푼다(torch.linalg.lstsq). W★ = argminW ‖ F̃ W − Y ‖²F ⇒ 정규방정식 (F̃⊤F̃) W = F̃⊤Y
  4. 되써넣기: weight = (W★:mS)⊤, bias = W★마지막행을 student 마지막 선형층에 복사한다. SGD·epoch 없음 — 한 번의 대수 연산.
정리 (같은 답의 정확한 조건). teacher 출력맵이 student 특징의 아핀 함수로 표현 가능하면, 즉 어떤 (W,b)에 대해 fT(x) = W·φS(x) + b가 모든 x에서 성립하면, 위 lstsq 해는 그 (W,b)를 정확히 복원한다. 이는 rank(F̃) ≥ rank([ 목표를 설명하는 데 필요한 방향 ]), 실무적으로 student 마지막 hidden 폭 mS ≥ teacher 출력맵의 rank일 때 성립한다. 폭이 모자라면(bottleneck) 잔차가 0이 될 수 없다 — §6이 그 경계의 실측.

선형 teacher면 φS가 곧 선형이라 이 조건이 그대로 만족되어 held-out top-1 = 1.0, MSE ≈ 0(데모 8e-15). 비선형 teacher면 φS(student의 비선형 특징)가 teacher와 다른 기저라 head-only로는 부족하고 §7의 은닉 정합이 필요하다.

6용량 조건 — 보장의 정확한 경계

같은 답은 항상 보장되지 않는다. student의 마지막 hidden 폭이 teacher 출력 맵의 rank를 선형 재구성할 만큼 넓어야 한다. 좁으면(bottleneck) 수학적으로 불가능하며, Ferry는 이를 숨기지 않고 잔차로 보고한다.

데모 실측(held-out top-1 일치율, teacher MLP[32,128,96,64,10], out_dim=10). 폭이 임계 rank를 넘는 순간 100% 일치 + MSE≈4e-15로 급격히 도달한다. 그 전까지는 bottleneck.

7비선형 teacher — 은닉층 정합으로 지원

teacher가 진짜 비선형(ActMLP, ReLU·GELU·tanh)이면 마지막 선형층만 맞추는 것(head-only)으로는 부족하다 — student의 비선형 특징이 teacher와 다른 기저라 held-out이 1.0에 못 미친다. Ferry의 stage 2b align_hidden은 그 기저 자체를 바꾼다: forward sweep으로 student 각 은닉층의 pre-activation을 teacher의 대응 pre-activation에 closed-form 회귀 (lstsq)시킨 뒤 head를 맞춘다. 여전히 데이터 없음·gradient loop 없음·결정적 대수 — 합성 probe만 쓴다.

그리고 stage 3 distill(§9)을 warm start로 이어 붙이면 남은 잔차까지 거의 닫힌다 — 아래 표의 마지막 열.

student 깊이활성head-onlyhidden+head+distill해석

gelu0.7520.9710.992사실상 일치

데모 실측(held-out top-1): teacher ActMLP[32,96,64,10], student ActMLP[32,128,96,10]. 닫힌 형식 은닉 정합만으로 ~0.97로 도약하고, warm-started gradient distillation이 ~0.99로 마무리한다. 즉 비선형 한계는 닫힌 형식의 벽이 아니라 gradient를 허용하면 닫히는 경계였다. TinyLM(트랜스포머)은 flat linear chain이 아니라 은닉 정합 범위 밖이며 head-only + distill로 다룬다(§8).

7.1 forward sweep 수식

student의 ℓ번째 은닉 선형층을 teacher의 대응 층 pre-activation에 맞춘다. student feature hS(ℓ-1)를 입력으로, teacher의 pre-activation zT(ℓ)를 목표로 같은 lstsq를 푼다:

목표: zS(ℓ) = W(ℓ) hS(ℓ-1) + b(ℓ) ≈ zT(ℓ) (폭 다르면 crop/0-pad) 해: [ W(ℓ) ; b(ℓ) ] = argmin ‖ [hS(ℓ-1)|𝟏]·W − zT(ℓ) ‖²F for ℓ = 1 … L'−1
  1. 전방 진행: ℓ을 입력→출력 순서로 sweep. 각 단계는 이미 정합된 이전 층이 만든 hS(ℓ-1)를 입력으로 쓴다(그래서 forward sweep).
  2. 층별 회귀: 위 식을 §5와 동일한 닫힌 형식 lstsq로 푼다. teacher 폭이 더 넓으면 목표를 student 폭으로 잘라(crop) 맞추고, 좁으면 0-pad한다.
  3. head 마무리: 모든 은닉층을 맞춘 뒤 §5의 align_output으로 마지막 층까지 맞춘다.

이로써 student의 내부 기저 φS가 teacher의 표현을 따라가게 되어 §5 정리의 조건(teacher 출력이 student 특징의 아핀 함수)이 회복된다 — 깊이가 같을 때 ~0.97. 깊이가 모자라면(층 수 부족) 추적할 teacher 층이 더 많아 완전히 닫히지 않는다(부분 한계).

8LLM 같은 모델 — 자기회귀 오차 누적

Ferry의 2단계는 작은 GPT형 트랜스포머(TinyLM: token/pos embedding + 멀티헤드 self-attention + GELU MLP + LayerNorm + LM head)에도 구조적으로 그대로 적용된다. stage 1은 attention q/k/v/o·MLP·embedding·head를 이름으로 전부 전이하고 (데모: 30개 텐서, SvdProject 11 + CropPad 18 + Copy 1, coverage 100%), stage 2는 LM head를 토큰 위치별로 다시 맞춘다. 외부 데이터 대신 랜덤 토큰 시퀀스 probe만 쓴다.

9서로 다른 vocabulary — 병합 전 어휘 정합 (stage 0)

LLM의 출력 축은 그 모델 자신의 vocabulary다. teacher와 student가 다른 tokenizer로 만들어졌다면 LM-head 너비가 다르고(예: Vt=64 vs Vs=48), 같은 토큰 id j가 서로 다른 토큰을 뜻한다. 그래서 logits를 위치별로 바로 비교하거나 정합하는 것은 무의미하다 — 병합 전에 어휘를 먼저 맞춰야 한다.

Ferry는 reconcile_vocab으로 VocabMap을 만든다: t_for_s(student id → teacher id, -1은 student 전용 토큰)와 (Vt×Vs) 선택 행렬 projection. 두 연산 — remap_ids는 student-공간 probe를 teacher id로 번역해 같은 probe를 양쪽에 먹이고, project는 teacher logits (··,Vt)를 student 어휘 공간 (··,Vs)로 사상한다(student 전용 열은 0 = teacher 신호 없음). 이후 agreement/align_output/distill에 vocab_map=만 넘기면 나머지 파이프라인은 동일하게 돈다.

선택 행렬: S ∈ {0,1}Vt×Vs, S[ t_for_s[j], j ] = 1 if t_for_s[j] ≥ 0; else 열 j = 0 project: ŷS = yT · S (teacher logits (··,Vt) → student 공간 (··,Vs)) remap_ids: ĩ = max( t_for_s[i], 0 ) (student probe id i → teacher id; −1은 0으로 clamp)

즉 같은 probe를 remap_ids로 양쪽에 먹이고, teacher logits를 S로 student 어휘에 정렬한 뒤 §5의 lstsq를 그대로 적용한다. S의 각 열은 one-hot(매핑된 토큰) 또는 0(student 전용) — 따라서 student 전용 토큰은 teacher 타깃이 0이 되어 학습 신호가 없다.

기본 t_for_s는 shared-prefix 맵(arange(min(Vt,Vs)), 꼬리는 -1)이다 — 실제 배포에서는 tokenizer 문자열을 매칭한 맵을 넘겨야 한다. vocab_map=None은 엄격한 no-op이라 같은 어휘를 쓰는 기존 MLP/TinyLM 동작과 테스트는 그대로다.

현실적 최악 — 세 축이 동시에 다를 때

위 표는 vocabulary 한 축만 다른 경우다. 실제 두 모델은 보통 LM-head/vocabulary · 중간층 depth · hidden width가 모두 다르고, vocabulary 맵도 깔끔한 shared-prefix가 아니라 뒤섞이고 부분적(임의 위치·일부만 매핑)이다. _demo_combined_mismatch는 이를 한꺼번에 쌓는다: teacher TinyLM(V=72,dim=80,L=4) → student (V=48,dim=40,L=2), 그리고 _scrambled_vocab_map(40/48만 임의 슬롯에 매핑).

10닫힌 형식의 한계를 닫다 — gradient distillation

§7·§8의 잔차는 닫힌 형식 방법의 한계였지 함수의 한계가 아니었다. 보편 근사 정리상 충분히 넓고 깊은 student는 비선형 teacher를 표현할 수 있다 — 닫힌 형식 lstsq가 그 해에 닿지 못했을 뿐이다. stage 3 distill은 제약을 하나 푼다: gradient를 허용한다. 그래도 데이터는 여전히 없다.

핵심은 매 step 새 합성 probe를 뽑아 teacher의 출력을 타깃으로 MSE를 최소화하는 것이다. 고정 probe를 닫힌 형식으로 풀면 그 probe에 과적합되어 held-out이 오르지 않는다(이전 실험에서 확인). 반면 매번 새로 뽑는 무한 합성 probe는 입력 분포 전체를 덮어 일반화된다. 닫힌 형식 stage 1–2를 warm start로 두면 수렴도 빠르다.

진짜 목적(기대위험): minθS 𝔼x∼P ‖ fS(x; θS) − fT(x) ‖² step t의 추정: gt = ∇θS ‖ fS(Xt) − fT(Xt) ‖², Xt ∼ P (매 step 새로 추출) 고정 probe(나쁨): minθS ‖ fS(Xfix) − fT(Xfix) ‖² → Xfix에만 과적합
  1. warm start: stage 1–2의 닫힌 형식 해를 θS 초기값으로 둔다(이미 teacher 근처).
  2. 신선한 probe: 매 step Xt = synthetic_probe/token_probe(seed+1+t)로 새로 뽑는다 — 무한 probe 스트림이 분포 P를 표본한다.
  3. teacher 타깃: Yt = fT(Xt)를 no_grad로 구한다(데이터 아님, teacher가 라벨).
  4. Adam step: MSE의 gradient gt로 θS를 갱신. held-out probe(seed−99)로 일반화 측정.

고정 probe의 닫힌 형식 해는 그 probe에서 잔차 0을 만들 수 있어도 held-out에서 실패한다(과적합). 매 step 새 probe를 뽑는 SGD가 기대위험 자체를 최소화하므로 임의 입력에 일반화된다 — 이것이 stage 3가 stage 2의 잔차를 닫는 메커니즘이다.

결과: 깊이가 맞고 충분히 넓은 비선형 student는 warm-started distillation 후 held-out top-1 ~0.99(relu/gelu/tanh .996/.992/.996, §7). TinyLM은 per-token top-1 0.41→0.89, 생성 감쇠 곡선이 평탄해진다(§8).

데이터-free 보장은 유지된다: distill은 두 모델과 probe 차원만 받는다 — 데이터셋·파일·tokenizer 없음. in_dim(연속) 또는 vocab+seq(토큰) 중 정확히 하나의 입력 모드만 고른다. stage 1–2는 여전히 결정적 닫힌 형식이고, gradient loop은 stage 3에만 격리되어 있다.

11정직한 결론

보장하는 것: 용량 조건이 충족되면, 외부 데이터 없이 합성 probe만으로 student를 teacher와 임의 입력에서 동일하게 만든다. 선형 teacher는 닫힌 형식만으로 held-out top-1 100%(MSE≈6e-15), 비선형 teacher는 닫힌 형식+distill로 ~0.99까지 닫힌다.
경계 — 종류가 다르다. (1) student가 너무 좁으면(bottleneck) 같은 답은 수학적으로 불가능하다 — 근본 벽이지만 폭을 늘리면 제거된다(§6, w48=100%). (2) teacher 비선형은 더 이상 벽이 아니다: 닫힌 형식 은닉 정합(§7)이 ~0.97, stage 3 distill(§9)이 ~0.99로 닫는다 — gradient 제약을 푼 결과다. (3) LLM-급 자기회귀 생성의 잔차 누적은 distill로 크게 평탄해지지만(step8 .34→.61) 깊은 트랜스포머·유한 step 탓에 horizon에 따라 여전히 증가한다 — 평탄화이지 소거는 아니다(§8). 모든 경우 Ferry는 가짜 일치를 만들지 않고 잔차를 그대로 보고한다. 트랜스포머 전층 정합과 permutation 정렬은 향후 과제다.

그래서 Ferry는 "같은 답"을 조건부로 보장한다 — 가능한 곳에서는 정확히 맞추고, 한계가 있는 곳에서는 그 한계가 근본 벽인지(폭) / 제약을 풀면 닫히는지(gradient) / 평탄화만 되는지(자기회귀)를 정직하게 구분해 드러낸다.

II실모델 확장 — Gemma-2 → Aster

§0–11은 toy core(MLP·ActMLP·TinyLM, 합성 teacher)에서 같은-답 보장의 조건을 증명했다. Part II는 같은 파이프라인을 진짜 사전학습 LLM에 적용한다: teacher = 실제 Gemma-2-2B/9B, student = 구조가 바뀐 aster-1b(Rust from-scratch 한국어-우선 SLM, 학습 진행 중인 실모델). toy의 깔끔한 보장이 현실 제약(거대 vocabulary·head_dim 불일치·압축 용량·tokenizer 격차) 아래에서 어디까지 가는지를 데이터-free·CPU·결정적으로 정직하게 측정한다.

핵심 발견(미리): 실모델에서는 toy의 "정답 일치"가 아니라 도달(reachability) ✅ vs 정렬(alignment) ❌로 갈라진다. 한국어를 분포에 살려내는 것은 데이터-free 닫힌 형식으로 가능했지만, 의미·문법 정렬은 학습(KD)의 몫으로 남는다 — §15.

14도달 ≠ 정렬 — 닫힌 형식 직교 Procrustes (b′)

byte-comp seed는 Gemma embed의 고유 특이기저에 산다 — Aster의 hidden 기하와 회전이 어긋나 h·row 순위가 노이즈가 되고 질량이 흩어진다. §7의 은닉 정합 정신으로, 데이터 없이·gradient 없이 그 회전만 닫힌 형식으로 맞춘다.

  1. 앵커 수집: whole-token 매칭 19880개(한글 1826)를 앵커로. 합성 shared-probe를 student에 흘려 tied-head 입력 특징 F = final_hidden을 잡고, 같은 probe를 remap해 실제 Gemma-2B teacher의 앵커 logit 열을 모은다.
  2. 목표 embed 회귀: 정규방정식 A=F⊤F, B=F⊤Twhole, Efit = (A+λI)−1B — 앵커가 가져야 할 embed(Aster 출력 기저)를 lstsq로 구한다.
  3. 직교 Procrustes: seed와 Efit 사이 회전만 학습한다. M = Eseed⊤Efit = UΣV⊤, R = U V⊤, Ealigned = Eseed R
  4. 전 row에 전이: 같은 proj에서 나온 byte-composed 한글 row도 같은 기저에 살므로 R을 48000개 전부에 적용. 직교 R이라 row-norm을 정확히 보존(한글 경쟁력 유지).

15실모델의 정직한 결론 — 도달했으나, 정렬·유창은 학습의 몫

마지막으로 §10의 distill(데이터-free gradient KD)을 실제 Aster에 적용해 닫힌 형식 너머의 천장을 측정한다. teacher = 실제 Gemma-2B, 매 step 신선한 토큰 probe, teacher logits를 VocabMap으로 student 어휘에 투영해 타깃으로 쓴다.

정합 전 (embed-map 전이본)0.000109.95−0.080
distill 후 (100 step)0.00010.720.769

방향성(cosine −0.08→0.77)·크기(MSE 10×↓)는 크게 개선되지만 top-1은 probe 위에서도 0. 원인: KD가 랜덤 토큰(실 텍스트와 극심히 off-distribution)에서만 teacher를 모방 → 실 프롬프트 greedy로 전이되지 않는다. toy의 TinyLM이 0.89까지 닫힌 것과 달리(§8), 용량 ~17%·vocab 41%· tied-embed 미정렬이 겹친 실모델은 data-free로는 천장이 낮다(Qwen 유사 실험도 0.20에서 멈춤).

도달(reachability)은 데이터-free 닫힌 형식으로 달성. embed=0 collapse → vocab-map → byte-composition으로 한국어를 분포에 살려냈고(질량 2%→91%, 샘플링 0/30→30/30), 직교 Procrustes 정렬이 top-k 생존까지 끌어올렸다(rank 95→9, Rust 기본 top-k40에서 한글 생성).
정렬(alignment)·유창성(fluency)은 학습의 몫. 닫힌 형식으로 닫을 수 있는 건 hidden-basis 회전 어긋남뿐이다. (1) byte order 손실(byte 평균 = crude init), (2) 문법·순차 신호 부재는 가중치 대수로 만들 수 없다 — 이는 on-distribution 학습/KD의 영역이고, 그것은 Ferry의 데이터-free 제약을 위반한다(별도 승인 사항). 비유하면 도달 = 혈류 연결(살아있음), 정렬 = 신경 배선(협응) — 후자는 이식이 아니라 재활(학습)로 생긴다.

그래서 실모델 확장의 정직한 답: 구조가 바뀐(17% 용량·다른 vocab) 실제 LLM으로의 전이는 데이터-free로 "한국어 도달"까지 가능하지만, "유창한 한국어"는 학습 없이는 불가능하다. Ferry는 이 경계를 — 무엇이 닫혔고(도달) 무엇이 남았는지(정렬·유창) — 가짜 성공 없이 수치로 구분해 드러낸다.
